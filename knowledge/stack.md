# Bohdan's Tech Stack and Preferences

## Frontend

**React / Next.js** — Bohdan's primary framework for production web apps. Prefers Next.js App Router for server components, SSG, and API routes in one project.

**TypeScript** — Used on all new projects. Strong typing catches bugs at compile time and improves team collaboration.

**Vanilla CSS / CSS Modules** — Prefers writing CSS directly with custom properties for design tokens rather than utility-first frameworks. Knows Tailwind CSS as well.

**Accessibility** — All UI is built to WCAG AA standard. Uses semantic HTML, ARIA attributes, and keyboard navigation as defaults, not afterthoughts.

## Backend

**Node.js** — Used for JavaScript-heavy backends, real-time services (WebSockets), and API gateways.

**Go** — Preferred for high-performance, high-throughput services. Used in the REST API Service project for its speed and simplicity.

**REST APIs** — Primary API style. Designed with clear resource semantics, consistent error responses, and OpenAPI documentation.

## Databases

**PostgreSQL** — Primary database for all relational data. Used with Supabase for managed hosting.

**Redis** — Used for caching, rate limiting, and session storage.

**Supabase** — Full Postgres-as-a-service with auth, real-time, and storage. Powers this portfolio's comments and AI features.

## Infrastructure and Tooling

**Docker** — Used for local development environments and containerised deployments.

**Vercel** — Preferred hosting for Next.js projects. Zero config, excellent DX, global edge network.

**GitHub Actions** — CI/CD pipelines for running tests and deploying on merge.

**Figma** — Used for design collaboration, wireframing, and building design systems.
